# Story Title Guidelines

## Format
**Title**: [Brief, action-oriented description]

## Rules
- Start with action verb
- Keep under 50 characters
- Describe the capability, not implementation
- Use present tense

## Examples

**Good Titles:**
- View Project Dashboard
- Create User Story
- Export Test Results
- Assign Story Components
- Generate Code Documentation

**Avoid:**
- "User can view dashboard" (too verbose)
- "Dashboard" (not action-oriented)
- "Implement dashboard viewing functionality" (implementation-focused)
- "As a user I want to view the dashboard" (full story in title)

## Pattern
[Verb] + [Object] + [Context if needed]

- **View** Project Dashboard
- **Create** User Story
- **Export** Test Results
- **Assign** Story Components
