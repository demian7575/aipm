# User Role Guidelines

## Format
**As a** [user type/role]

## Common Roles

**Primary Users:**
- project manager
- developer
- tester
- stakeholder
- team lead

**System Users:**
- system administrator
- end user
- API consumer
- data analyst

**External Users:**
- client
- auditor
- reviewer
- external system

## Rules
- Use lowercase
- Be specific but not overly detailed
- Focus on the role's relationship to the feature
- Avoid job titles when role is clearer

## Examples

**Good:**
- As a **project manager**
- As a **developer**
- As a **system administrator**
- As a **API consumer**

**Avoid:**
- As a **Senior Software Engineer Level III** (too specific)
- As a **person** (too generic)
- As a **user of the system** (redundant)
