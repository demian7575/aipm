# Business Value Guidelines

## Format
**So that** [business value/benefit]

## Rules
- Explain the "why" behind the functionality
- Focus on outcomes, not activities
- Connect to business or user goals
- Avoid restating the functionality

## Value Categories

**Efficiency:**
- I can work more efficiently
- I can save time on routine tasks
- I can reduce manual effort

**Visibility:**
- I can track project progress
- I can identify bottlenecks quickly
- I can make informed decisions

**Quality:**
- I can ensure requirements are met
- I can maintain code standards
- I can reduce errors

**Collaboration:**
- the team can stay aligned
- stakeholders can provide feedback
- communication is improved

## Examples

**Good:**
- So that **I can track project progress at a glance**
- So that **I can make informed resource decisions**
- So that **the team can stay aligned on priorities**
- So that **I can ensure quality standards are met**

**Avoid:**
- So that **I can use the dashboard** (restates functionality)
- So that **the system works better** (vague)
- So that **it's easier** (no specific benefit)
