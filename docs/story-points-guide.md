# Story Points Guide

## Valid Values
**Story Points**: [1, 2, 3, 5, 8, 13, 21]

## Fibonacci Sequence
- Based on Fibonacci numbers for relative sizing
- Each value represents increasing complexity/effort
- Non-linear scale reflects estimation uncertainty

## Sizing Guidelines

### 1 Point - Trivial
- Simple text changes
- Configuration updates
- Minor bug fixes
- **Time**: 1-2 hours

### 2 Points - Small
- Simple form validation
- Basic UI adjustments
- Straightforward database queries
- **Time**: Half day

### 3 Points - Medium-Small
- New form creation
- Simple API endpoints
- Basic feature additions
- **Time**: 1 day

### 5 Points - Medium
- Complex forms with validation
- Multi-step workflows
- Integration with existing systems
- **Time**: 2-3 days

### 8 Points - Large
- New page/component creation
- Complex business logic
- Multiple system interactions
- **Time**: 1 week

### 13 Points - Very Large
- Major feature implementation
- Significant architectural changes
- Complex integrations
- **Time**: 1-2 weeks

### 21 Points - Epic
- Should be split into smaller stories
- Too large for single sprint
- High risk and complexity
- **Action**: Break down further

## AIPM System Rules
- Only these values accepted
- Required for all stories
- Used in workload calculations
- Validated during story creation
