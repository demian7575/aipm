# Given-When-Then Acceptance Criteria

```
Given the AI PM Mindmap application is installed
When a user seeds the backend
Then sample merge requests, stories, and acceptance tests are available for exploration

Given the outline tree view is open
When the user navigates with arrow keys or toggles nodes
Then the tree respects accessibility roles and persists expansion state

Given the mindmap view is open
When the user selects a merge request
Then the radial layout mirrors the structure of stories and tests

Given a story violates INVEST or contains ambiguous language
When it is displayed in the detail panel
Then corresponding warnings and guidance are shown
```
