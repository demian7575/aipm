# Role Assignment Guide

## Team Roles
- **Product Owner**: Defines requirements and priorities
- **Scrum Master**: Facilitates process and removes blockers
- **Developer**: Implements features and fixes
- **QA Engineer**: Tests and validates acceptance criteria
- **DevOps Engineer**: Manages infrastructure and deployments
- **UI/UX Designer**: Creates user interfaces and experiences

## Assignment Rules
1. Each user story must have exactly one assignee
2. Assignee email must be valid team member
3. Consider workload balance across components
4. Match skills to story requirements

## Component Areas
- **System (S/S)**: Core system functionality
- **WorkModel (WM)**: Business logic and workflows
- **DocumentIntelligence (DI)**: Document processing
- **Review & Governance (RG)**: Approval processes
- **Orchestration & Engagement (OE)**: User interactions
- **Run & Verify (RV)**: Testing and validation
- **Traceability & Insight (TI)**: Analytics and reporting

## Workload Distribution
Use Employee Heat Map to monitor:
- Percentage allocation per component
- Skill coverage across areas
- Bottlenecks and capacity gaps

## Assignment Process
1. Review story requirements
2. Check current team workload
3. Match skills to components
4. Assign to available team member
5. Update story status to Ready
