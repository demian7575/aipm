# Filter User Stories in Mindmap View

## Story Details
Add a filter button to the top toolbar that opens a modal allowing users to select filter criteria (status, component, assignee) to narrow down visible user stories in the mindmap view.

## Acceptance Criteria
Title: Filter User Stories in Mindmap View

As a: project manager
I want: to filter user stories by status, component, and assignee
So that: I can focus on specific subsets of stories and reduce visual clutter in the mindmap

Description: Add a filter button to the top toolbar that opens a modal allowing users to select filter criteria (status, component, assignee) to narrow down visible user stories in the mindmap view.

Components: WorkModel

Story Points: 5

Acceptance Tests:
1. Applying filters updates mindmap visibility
   Given: the filter modal is open with criteria selected
   When: I apply the filters
   Then: only user stories matching the selected criteria are visible in the mindmap view
2. Filter button opens modal with criteria
   Given: I am viewing the mindmap with multiple user stories
   When: I click the filter button in the top toolbar
   Then: a modal opens displaying filter options for status, component, and assignee
3. Display filtered non-root user story when selected
   Given: a non-root user story matches the filter criteria
   When: the user selects the filtered story
   Then: the story is displayed regardless of its position in the hierarchy
