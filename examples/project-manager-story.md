# Project Manager User Story Example

## Story Structure

**Title**: Track Team Workload

**Description**: 
As a project manager  
I want to view workload distribution across team members  
So that I can balance assignments and identify bottlenecks

**Story Points**: 5

**Status**: Ready

**Assignee**: pm@company.com

**Components**: 
- Traceability & Insight (TI)
- Orchestration & Engagement (OE)

## Acceptance Criteria

### Test 1: Workload Dashboard Access
```
Given I am logged in as a project manager
When I navigate to the team workload page
Then I see a list of all team members with their assigned stories
```

### Test 2: Workload Visualization
```
Given I am viewing the team workload dashboard
When the page loads
Then I see each team member's story points total and percentage distribution
```

### Test 3: Component Breakdown
```
Given I am on the workload dashboard
When I select a team member
Then I see their workload broken down by component categories
```

### Test 4: Overload Identification
```
Given a team member has more than 21 story points assigned
When I view the workload dashboard
Then their name is highlighted in red as overloaded
```

## Project Manager Context
- Focuses on team management and resource allocation
- Emphasizes visibility and decision-making capabilities
- Addresses common PM pain points (workload balancing)
- Provides actionable insights for project planning

## Role-Specific Value
- Enables data-driven assignment decisions
- Helps prevent team member burnout
- Supports sprint planning activities
- Improves project delivery predictability
